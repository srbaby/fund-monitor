## 原理

```
乐咕口径恒等式：滚动PE = ∑总市值/∑滚动净利润，盈利盘中不变
  → 盘中真PE = 昨夜官方PE × (实时总市值 / 昨收总市值)
  → 百分位   = 全量历史排序数组二分查找（精确ECDF，零插值误差）
消灭两大误差源（2026-06-10实测偏差1.25pp = β弹性0.86 + 线性插值0.38）
```

## 验证期运行纪要（2026-06-11 会话固化）

**实测结论**

- 腾讯 `qt.gtimg.cn` 指数级总市值字段（f[45]）盘中实时：双采样 ts 偏差≤2秒、随快照逐笔跳动、与点位变动不等比（独立聚合）。叠加看板10秒轮询，端到端陈旧≤15秒，对百分位精度可忽略。
- GitHub Actions cron 延时是常态：06-10 的 22:30 槽实际 01:41 执行（延 3h11m）。双槽+幂等跳过设计有效；`log[]` 的 `delayMin` 字段持续量化。
- 首晚 `log[]` 为空属设计内必然（第一晚只立锚）；第一条 errPp 自 06-11 晚起累积，两周KPI判定期 ≈ 至 06-24。

**已知bug（转正前修，验证期手动规避）**

- `pe_nightly.py` 日期检查 `last_date != today_bj`：槽位延迟跨午夜后（含手动触发！）会 exit 1 拒写当晚有效锚。06-10 晚靠手动运行掩盖。修法：放宽为"last_date ∈ {今天, 昨天(若现在<凌晨阈值)}"。

**数据源研究固化（2026-06-11 下午会话）**

- **中证指数官网 JSON 端点实测可用**（AI/脚本均可直拉）：
  `https://www.csindex.com.cn/csindex-home/perf/index-perf?indexCode=000300&startDate=YYYYMMDD&endDate=YYYYMMDD`
  返回逐日 OHLCV+`peg`(市盈率)。OHLCV 自 2005 发布日全量；**peg 自 2012-01~2013-06 之间起有值（深度≈13年，精确起点待回拉确定）**。
- **实证：腾讯 qt 的 PE 字段(f[39]) ≡ 中证官方 peg**（06-10收盘 14.30 vs 14.31，舍入差）。腾讯=官方口径的秒级实时转发，绝对值可信度升级。
- 官方 peg 14.31 vs 乐咕滚动 13.56（差5.5%）：盈利口径不同（静态/滚动/负值处理，**待办：本地对照乐咕序列判定**，年报季跳变可甄别）。两家绝对值不可互查对方分布；各自自洽算百分位则因排序不变性可比。
- **数据备胎定位（腾讯+中证官网）**：乐咕仍是主线，且每日全量拉取、20年序列始终在手，无单日断供之虞。万一乐咕数据源消失：用中证官网序列与手中乐咕历史做**映射匹配**（同窗对照标定口径转换），把官方数据扩展到20年口径作为替代主线——具体映射方法**后面专题研究**（候选素材：同窗百分位对照、口径比值序列、腾讯mcap链式衔接断点）。
- 腾讯接口请求纪律：必须带随机参数防CDN陈旧缓存（实测裸请求返回过5月21日缓存）；看板现行10秒单标的轮询远低于风控阈值。

**标签语义提示**
